## 1. Посекционное сравнение

### 1.1 Header / Навигация

| | Текущий сайт | Редизайн |
|---|---|---|
| **Логотип** | Слева, маленький | По центру, крупнее |
| **Навигация** | Текстовые ссылки справа | Полностью спрятана за бургер-меню |
| **Переключатель языка** | Не виден на скриншоте | UA / EN — справа от логотипа |
| **CTA в хедере** | Нет | Нет (упущение!) |
| **Фон** | Тёмный navy | Белый с blur-эффектом |

**Проблема редизайна:** Бургер-меню на десктопе — спорное решение для B2B продукта. Enterprise-клиенты привыкли видеть навигацию сразу. Ссылки «Модулі», «Ціни», «Демо» должны быть на виду.

**Рекомендация:** Вернуть видимую навигацию на десктопе. Бургер — только для мобилки. Добавить CTA-кнопку «Замовити демо» справа.

### 1.4 Основні модулі системи

| | Текущий сайт | Редизайн |
|---|---|---|
| **Layout** | Круговая диаграмма: модули вокруг центрального лого ProfITsoft | Сетка 4×2 из карточек с иконками |
| **Модули** | Продажі, Андеррайтинг, Комісійна, Врегулювання, Фінмоніторинг, ДМС, Перестрахування, Фінанси | Те же 8 модулей |

**Контентный конфликт:** В текущем сайте модули показаны как связанная система (hub-and-spoke). В редизайне — как разрозненные карточки. Теряется визуальное сообщение «это единая платформа».

**В HTML редизайна** есть закомментированная альтернатива — `modules-circle` (круговая диаграмма с орбитами). Она лучше текущего варианта с сеткой карточек, но выглядит сложновато.

**Рекомендация:** Оставить карточки, НО добавить визуальный элемент связности — например, тонкие линии между карточками или центральный логотип. Либо раскомментировать circular diagram, упростив анимации.

**⚠️ ПРОБЛЕМНАЯ СЕКЦИЯ (flagged клиентом)**

В редизайне: карточки с синей левой полоской (`border-left: 4px solid`) расположены вертикально. Это та самая «синяя вертикальная линия + скруглённые углы», которую клиент отметил как AI-образную.

**Почему выглядит AI-сгенерированной:**
- Паттерн `border-left + icon + text` — буквально дефолтный компонент из любого UI-кита
- Стоковое фото серверной — стерильный generic
- Теги-пилюли (Java, Spring Boot) — шаблонный элемент, который ничего не добавляет
- Hover-эффект `translateX(10px)` — типичный для AI-сгенерированного CSS

## 5. AI-образные секции

### 5.1 Технічна архітектура — 🔴 Высокий уровень «AI-шаблонности»

**Что именно выглядит AI-generated:**
- Вертикальный список с `border-left: 4px solid var(--royal-blue)` — это дефолтный компонент из Tailwind UI / Chakra
- Каждый элемент: иконка (FontAwesome) + текст в одну строку = шаблон
- Hover `translateX(10px)` = одно и то же у ChatGPT и Cursor
- Теги-пилюли (Java, Spring Boot, Docker) = визуальный мусор, не добавляют ценности
- Стоковое фото серверной комнаты справа — generic как из брошюры 2015 года

### 5.2 Історія продукту — 🔴 Высокий уровень «AI-шаблонности»

**Что именно выглядит AI-generated:**
- Вертикальная синяя линия 3px + круглые точки = классический Bootstrap/Tailwind timeline
- Карточки справа от линии с `border-left: 4px solid` = двойное повторение паттерна
- Hover `translateX(5px)` = тот же эффект, что в архитектуре
- Год слева, контент справа, линия посередине = буквально первый результат по запросу «timeline CSS»

### 5.3 Pricing — 🟡 Средний уровень

**Что шаблонно:**
- Две карточки рядом с featured/обычная — самый базовый pricing layout
- Чеклист с галочками = в каждом шаблоне
- `featured` карточка с толстой синей рамкой = стандарт

**Но:** Для pricing-секции шаблонность допустима — пользователи ожидают привычный формат.

### 5.4 Модулі (карточки) — 🟡 Средний уровень

**Что шаблонно:**
- Сетка 4×2 одинаковых карточек с иконкой + заголовком + описанием
- Hover с `translateY(-10px) + shadow` = universal template hover
- `::before` полоска сверху при hover

### 5.5 Feature zigzag — 🟡 Средний уровень

- Чередование текст-лево/фото-право и наоборот — паттерн из каждого landing page template
- `feature-badge` пилюли = Vercel/Linear клон
- Чеклист с кружками-галочками

### 5.6 Другие шаблонные элементы

- **News cards** — 3 карточки с фото сверху, бейджем, датой — 100% стандарт
- **Client logos** — бесконечный скролл в grayscale — стандартный паттерн
- **Training stats** — три квадрата со статистикой — стандарт

## 6. Альтернативы для проблемных секций

### 6.1 Технічна архітектура — Альтернативы

**Вариант A: Layered Architecture Diagram (вдохновение: Stripe)**

Интерактивная схема слоёв:
- Горизонтальные полосы/слои, стоящие друг на друге (как слои торта)
- Снизу вверх: `MS SQL Server` → `Java / Spring Boot Backend` → `REST API` → `Web Frontend`
- Каждый слой — полупрозрачный стеклянный блок с лёгким градиентом (glassmorphism)
- При hover на слой — он «выдвигается» вперёд, показывая описание
- Цвета: от тёмно-синего внизу к светлому вверху
- Без стокового фото вообще — сама схема и есть визуал
- Шрифт: monospace для технических терминов (как у Vercel)

```
┌─────────────────────────────────┐
│   🌐  Web Interface             │  ← светло-голубой
├─────────────────────────────────┤
│   ⚡ REST API Layer             │  ← средний синий
├─────────────────────────────────┤
│   ☕ Java / Spring Boot         │  ← тёмный синий
├─────────────────────────────────┤
│   🗄️  MS SQL Server             │  ← navy
└─────────────────────────────────┘
```

**Вариант B: Terminal / Code View (вдохновление: Linear, Raycast)**

Сделать секцию в стиле терминала / code editor:
- Тёмный фон (navy/near-black) — выделяется на странице
- Слева — «терминальное окно» с анимированным текстом:
  ```
  $ system --info
  ├── frontend: React Web App
  ├── backend:  Java 17 + Spring Boot 3
  ├── database: MS SQL Server
  ├── api:      RESTful, OpenAPI 3.0
  ├── deploy:   Docker, Kubernetes
  └── scale:    Horizontal auto-scaling
  ```
- Справа — краткий текст с 2-3 ключевыми техническими преимуществами
- Моноширинный шрифт + зелёный/голубой цвет текста = «мы настоящие разработчики»
- Это сразу убирает «AI-шаблонность» и добавляет техническую аутентичность

**Вариант C: Isometric Block Diagram**

- Изометрические 3D-блоки, связанные стрелками
- Каждый блок — компонент системы с иконкой
- CSS-only изометрия (transform: rotateX + rotateY) или SVG
- Мягкие тени + анимация появления при скролле
- Вдохновление: Cloudflare architecture page

### 6.2 Історія продукту — Альтернативы

**Вариант A: Горизонтальный scrollable timeline (вдохновление: Apple product pages)**

- Полноэкранная горизонтальная полоса
- Скролл страницы = горизонтальное перемещение по timeline (scroll-driven animation)
- Каждая версия — крупная карточка (300-400px шириной) с:
  - Большим номером версии (например, «3.0» крупно)
  - Название фазы
  - 2-3 тега фич
  - Возможно, маленький скриншот UI того периода
- Фон меняется от тёмного (2006) к светлому (2024) — визуальная метафора прогресса
- Нет вертикальной синей линии вообще!

**Вариант B: Карточки-вехи с counter-animation (вдохновление: Stripe Atlas)**

- Горизонтальный ряд из 6 карточек (не вертикальный!)
- Каждая карточка:
  - Крупный год (`2006`) с counter-анимацией при появлении
  - Маленький бейдж версии (`v1.0`)
  - Название + 1 строка описания
  - Accent-цвет карточки меняется (градиент от серого к синему по мере приближения к 2024)
- Между карточками — тонкие соединительные линии (не вертикальные!)
- На мобилке — swipeable горизонтальный скролл

```
[2006]----[2009]----[2013]----[2019]----[2021]----[2024]
 v1.0      v2.0      v3.0      v4.0      v5.0      v6.0
```

**Вариант C: Версионная сетка (вдохновление: GitHub releases)**

- Не timeline, а сетка 3×2 из карточек
- Каждая карточка = версия:
  - Крупный badge `v6.0` (последняя — выделена цветом)
  - Год как подписка
  - Заголовок
  - Список фич как `<code>` теги (как GitHub tags)
- Самая свежая версия — крупнее остальных (2:1 на первой строке)
- Выглядит как changelog, а не как хронология — более техничный вид

## 7. Общие рекомендации

### 7.1 🔴 Критично — стоковые изображения

**Главная проблема всего редизайна:** Все фотографии — стоки с Unsplash.

В HTML прямые ссылки:
- `unsplash.com/photo-1551434678-e076c223a692` (hero)
- `unsplash.com/photo-1460925895917-afdab827c52f` (youtube)
- `unsplash.com/photo-1554224155-6726b3ff858f` (youtube)
- `unsplash.com/photo-1558494949-ef010cbdcc31` (архитектура)
- и ещё ~10 штук

**Что нужно:**
1. Hero: скриншот продукта в device mockup ИЛИ фото реальной команды
2. Фичи: реальные скриншоты UI (с текущего сайта — там они есть!)
3. YouTube: реальные thumbnail'ы с канала
4. Архитектура: кастомная диаграмма (не фото)
5. Training: фото с реального обучения или скриншот курса
6. News: реальные изображения из статей

### 7.2 🔴 Критично — потерянные секции

Вернуть в «Ключові характеристики»:
- Автоматична система нарахувань
- Інтеграції
- Звіти
- ІМКД (Інтернет-магазин та Клієнтський доступ)

Это реальные фичи продукта, которые клиенты ищут. Их нельзя было убирать.

### 7.3 🟡 Навигация на десктопе

Вернуть видимые ссылки в хедер:
```
[Burger] [Модулі] [Характеристики] [Ціни] [Відео]   [Logo]   [UA|EN] [Замовити демо ➜]
```
Бургер можно оставить для полного списка, но top-5 ссылок должны быть видны.

### 7.4 🟡 Визуальная монотонность

Все секции используют одинаковый визуальный ритм:
- Белый фон → серый фон → белый фон → серый фон
- Одна и та же палитра (синий + серый)
- Одинаковые rounded-corner карточки везде

**Как разбить:**
1. **Одну секцию сделать full-bleed тёмной** (navy/dark) — например, архитектуру или pricing. Это создаст визуальный «стоп-момент».
2. **Добавить gradient mesh или subtle pattern** в фон 1-2 секций
3. **Использовать разные форматы контента:** не только «текст + фото», но и таблицы, диаграммы, видео-embed

### 7.5 🟢 Хорошие решения, которые стоит сохранить

1. Floating-карточки в hero — отличный визуальный элемент
2. Двухцветный заголовок с синим акцентом
3. Клиентские логотипы с авто-скроллом
4. Scroll-анимации (но не переборщить)
5. Slide-out меню для мобилки
6. Pricing с зачёркнутой ценой
7. Секция «Новини» — показывает активность
8. Top banner

### 7.6 🟢 Что добавить для «вау-эффекта»

1. **Product demo video** в hero — 30-секундный GIF или авто-играющее видео вместо стоковой фотки
2. **Testimonial-секция** — цитаты от реальных клиентов (1-2 карточки). Enterprise-клиенты это обожают.
3. **Live demo CTA** — «Спробуйте демо-версію прямо зараз» с формой запроса
4. **Числа побольше и поконкретнее** — «Обробляємо 500 000+ полісів щороку» или «Зменшуємо час обробки збитків на 60%». Сейчас только «18+ років» и «11+ компаній» — можно добавить операционные метрики.
5. **Comparison table** — «КСАСК vs. ручна робота» или vs. конкурентов. Страховые компании любят таблицы.

### 7.7 Технические замечания по коду

1. **Весь CSS inline в HTML** — нужно вынести в отдельный файл
2. **~4600 строк в одном файле** — разбить на компоненты (если это будет Next.js/React)
3. **Unsplash URL с параметрами** (`w=600&h=400&fit=crop`) — это temporary, нужны локальные изображения
4. **Custom cursor** в JS — удалить, это раздражает и не работает на тач-устройствах
5. **Parallax эффект** на hero-shapes — OK на десктопе, проверить производительность на мобилке
6. **`page-loader`** — с анимацией загрузки. На быстром хостинге выглядит как задержка. Лучше убрать или сделать instant.
